MyFlix Movie API

Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Features

  • Return a list of all movies to the user
  • Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user
  • Return data about a genre (description) by name/title
  • Return data about a director (bio, birth year, death year) by name
  • Return data about movies by genre
  • Return data about movies by director
  • Return a list of all users
  • Return data about a single user
  • Allow new users to register
  • Allow users to add a movie to their list of favorites
  • Allow users to update their user info
  • Allow users to remove a movie from their list of favorites
  • Allow existing users to deregister

API Endpoint Documentation

Business Logic URL HTTP Method Query Parameters Request Body Data Format Response Body Data Format
Returns a list of all movies /movies GET - - A JSON object holding data about all the movies
Returns data about a single movie by title /movies/:title GET :title - A JSON object holding data about a single movie
Returns data about a genre by name /movies/genreInfo/:genre GET :genre - A JSON object holding data about a specific genre of movie
Returns data about a director by name /movies/directorInfo/:director GET :director - A JSON object holding data about a specific director
Returns data about movies by genre /movies/genre/:genre GET :genre - A JSON object holding data about movies of a certain genre
Returns data about movies by director /movies/directors/:director GET :director - A JSON object holding data about movies by a certain director
Returns a list of all users /users GET - - A JSON object holding data about all users
Returns data about a single user /users/:username GET :username - A JSON object holding data about a single user
Allows new users to register /users POST - A JSON object holding data about the user to add, structured like: {"name": "Tom", "password": "1234", "email": "tom@gmail.com", "birthdate": "1983-10-08"} A JSON object holding data about the the new user
Allows users to add a movie to their favorites /users/:username/movies/:movieID POST :username, :movieID - A JSON object holding data about the updated user
Allows users to update their user info /users/:username PUT :username A JSON object holding data about the user that was updated, structured like: {"name": "Tom", "password": "1234", "email": "tom@gmail.com", "birthdate": "1983-10-08"} A JSON object holding data about the updated user
Allows users to remove a movie from their favorites /users/:username/movies/:movieID DELETE :username, :movieID - A JSON object holding data about the updated user
Allows users to deregister /users/:username DELETE :username - A message confirming the user was deleted
